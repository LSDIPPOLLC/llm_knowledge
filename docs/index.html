Karpathy's LLM Wiki

Introduction

A personal knowledge base and agent memory system powered by an LLM wiki.

ideallm uses Andrej Karpathy's LLM Wiki pattern - a personal knowledge base where an AI compiles sources into a structured, interlinked wiki. The wiki sits between you and your raw sources, accumulating knowledge over time rather than rediscovering it on every query.

Two Purposes

  • Personal Knowledge Base - Tracking research, articles, papers, books, ideas
  • Agent Memory - Persistent memory across sessions; accumulates knowledge about projects, decisions, patterns

Core Principle

The raw/ directory is the source of truth. Wiki pages track their relationship to raw sources via versioned frontmatter. When raw changes, the wiki must be re-synced.

Quick Start

Get up and running with your personal wiki in minutes.

1

Open in Obsidian

obsidian /path/to/ideallm/wiki
2

Add sources to raw/

Add articles, papers, or notes to the raw directory.

3

Ingest content

> ingest raw/articles/filename.md
4

Query the wiki

> What does our wiki say about scaling laws?
5

Run lint to check health

> lint
6

Sync when sources change

> sync raw/articles/my-article.md

Project Structure

The wiki follows a simple but powerful directory convention.

ideallm/
├── CLAUDE.md              # Behavioral guidelines for the AI agent
├── WIKI_SCHEMA.md         # LLM Wiki schema (operations, conventions)
├── raw/                   # Source of truth (wiki must stay in sync with raw)
│   ├── articles/          # Web-clipped or manually added articles
│   ├── papers/            # Academic papers, technical reports
│   ├── notes/             # Personal notes, journal entries, reflections
│   └── assets/            # Downloaded images, figures, attachments
├── wiki/                  # LLM-generated wiki (YOU own this; LLM maintains it)
│   ├── index.md           # Master catalog of all wiki pages
│   ├── log.md             # Append-only activity timeline
│   ├── overview.md        # High-level synthesis of the knowledge base
│   ├── concepts/          # Concept pages
│   ├── entities/          # Entity pages (people, organizations)
│   ├── sources/           # Source summary pages
│   ├── comparisons/       # Comparison pages
│   └── reflections/       # Insights filed from queries
└── .claude/
    ├── memory/            # Lightweight session memory files
    ├── domains.md         # Topic-to-page index for quick discovery
    └── wiki_query.py      # Terminal search tool

Directory Purposes

raw/

Immutable source documents. NEVER modify directly.

wiki/

LLM-generated wiki. YOU own this; LLM maintains it.

.claude/

Session memory and search tools.

Frontmatter Schema

Every wiki page has YAML frontmatter for tracking metadata.

Wiki Page Frontmatter

---
title: Page Title
type: concept | entity | source-summary | comparison | reflection
sources: [list of raw/ files referenced]
related: [list of wiki pages linked]
created: YYYY-MM-DD
updated: YYYY-MM-DD
confidence: high | medium | low
status: current | stale | deprecated
raw_version: "1.0.0"
last_verified: YYYY-MM-DD
replaced_by: wiki/path/to/replacement.md  # for deprecated pages only
---

Status Values

current Wiki content is in sync with raw sources
stale Raw has changed since this page was last verified/ingested
deprecated This page is superseded; see replaced_by for current content

Raw File Frontmatter

---
title: Users API Reference
version: "2.1.0"
last_updated: 2026-04-15
---

The version field tracks when the raw content changes, enabling the sync workflow.

Wiki Operations

Core workflows for maintaining your wiki.

Ingest

> ingest raw/articles/my-article.md

Drop a file into raw/ and tell the AI to process it.

The AI will:

  1. Read the source
  2. Discuss key takeaways with you
  3. Create/update a summary page in wiki/sources/
  4. Update wiki/index.md with new page
  5. Update all relevant concept and entity pages
  6. Add links from existing pages that reference the new content
  7. Append an entry to wiki/log.md

Query

> What does our wiki say about scaling laws?

Ask questions and the AI synthesizes answers from the wiki.

The AI reads wiki/index.md, identifies relevant pages, synthesizes an answer with [[wiki-link]] citations.

Good answers can be filed back into the wiki as new pages.

Sync

> sync raw/articles/my-article.md

When raw sources change, sync the wiki to keep it current.

This:

  1. Detects version changes
  2. Marks affected pages as stale
  3. Re-ingests the changed source
  4. Deprecates old pages
  5. Updates frontmatter

Deprecate

> deprecate wiki/concepts/old-auth.md

Mark a page as superseded.

This adds status: deprecated, replaced_by frontmatter, a deprecation banner, and updates wiki/index.md.

Lint

> lint

Health-check for issues:

  • Contradictions between pages
  • Orphan pages with no inbound links
  • Missing pages for concepts mentioned but lacking their own page
  • Stale pages - pages with status: stale that need re-ingesting
  • Deprecated pages - superseded pages still linked

Search

Query your wiki from the terminal.

Wiki Query Tool

# Search all wiki content
python .claude/wiki_query.py "vibe coding"

# Search specific areas
python .claude/wiki_query.py "agentic engineering" --pages
python .claude/wiki_query.py "scaling laws" --sources
python .claude/wiki_query.py "karpathy" --recent

# Output as JSON
python .claude/wiki_query.py "moe" --json

Options

Option Description
<term> Search term (required, or pass via stdin)
--domains Search domain index only
--pages Search wiki pages only
--sources Search raw sources only
--recent Search recent activity in log
--json Output as JSON
