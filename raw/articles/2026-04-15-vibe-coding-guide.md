## What Is Vibe Coding?

Vibe coding is a software development practice where you describe what you want in plain language and let an AI model generate the code. The term was coined by **Andrej Karpathy** — co-founder of OpenAI and former AI leader at Tesla — on February 2, 2025.

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."
> — Andrej Karpathy, February 2, 2025

### The Key Distinction

Developer Simon Willison drew an important line: **"If an LLM wrote the code for you, and you then reviewed it, tested it thoroughly, and made sure you could explain how it works to someone else, that's not vibe coding — it's software development."**

Vibe coding specifically means accepting AI-generated code without fully reviewing it, relying on results and iterative prompting rather than deep code understanding.

## How Vibe Coding Actually Works

The vibe coding workflow has evolved far beyond copy-pasting prompts. Modern tools use **agentic workflows** that autonomously plan, execute, and verify:

### Step 1: Describe Your Intent
You describe what you want in natural language. This can be a voice command, a typed prompt, or even a screenshot of a design.

### Step 2: AI Plans the Architecture
Modern agentic tools analyze your codebase, understand dependencies, and plan multi-file changes.

### Step 3: Code Generation & Execution
The AI generates code, creates files, installs dependencies, and can even run tests to verify its own work. In 2026, the expectation has shifted to **full-stack generation**: frontend, backend, database, auth, API endpoints, and deployment — all from a single prompt.

### Step 4: Iterate on Results
You review the output, test the running application, and provide follow-up prompts to refine.

## Best Practices for Professional Vibe Coding

### 1. Always Review Critical Paths
Vibe code freely for UI components, boilerplate, and prototypes. But **always manually review** authentication, payment processing, data validation, and infrastructure code.

### 2. Write Tests Before You Vibe
The most effective vibe coders write tests first, then let the AI implement the code to pass those tests.

### 3. Use Structured Prompts
Vague prompts produce vague code. Be specific about frameworks, patterns, error handling, and edge cases.

### 4. Commit Frequently
AI agents can make sweeping changes. Commit after every successful feature so you have a clean rollback point.

### 5. Monitor Your Quota
Extended vibe coding sessions burn through tokens fast.

### Common Vulnerabilities in AI-Generated Code
- **SQL Injection & Input Validation:** AI code may return correct data in testing but fail to prevent injection attacks
- **Hardcoded Secrets:** AI frequently includes real API keys or test credentials directly in generated code
- **Deprecated Libraries:** Models trained on historical code can reintroduce deprecated protocols and weak cryptography
- **Hallucinated Dependencies:** AI sometimes suggests packages that don't exist, enabling typosquatting attacks
- **Buffer Overflows:** In C/C++, AI consistently generates unsafe functions like `scanf` and `strcpy` without bounds checking

## Vibe Coding: Pros and Cons

### The Good
- **Massive productivity gains:** 74% of developers report increased productivity; seniors see 81% improvement
- **Democratized software creation:** 63% of vibe coding users identify as non-developers, building real applications
- **Faster prototyping:** Full-stack apps from a single prompt, deployed in hours instead of weeks
- **Lower barrier to entry:** You can build a SaaS product without knowing React, SQL, or DevOps
- **Focus on product, not syntax:** Energy shifts from implementation details to user experience and business logic

### The Bad
- **Security vulnerabilities:** Nearly 1 in 4 AI-generated code snippets has a security flaw
- **Automation complacency:** When AI is consistently right, you stop checking — and miss the times it's wrong
- **Technical debt:** Code you don't understand is code you can't maintain
- **Debugging paradox:** 63% of developers have spent more time debugging AI code than writing it themselves at least once
- **Skill erosion:** 40% of junior developers admit deploying code without full understanding

## Beyond Vibe Coding: Agentic Engineering

Even Karpathy himself has moved on. In early 2026, he declared vibe coding "passé" and introduced a new term: **agentic engineering**.

> "'Agentic engineering': 'agentic' because the new default is that you are not writing the code directly 99% of the time, you are orchestrating agents who do and acting as oversight — 'engineering' to emphasize that there is an art & science and expertise to it."
> — Andrej Karpathy, 2026

The distinction matters: vibe coding is throwing prompts at an AI and accepting whatever comes back. Agentic engineering is **orchestrating teams of specialized agents**, defining quality gates, running automated tests, and maintaining human oversight at critical checkpoints.

Vibe coding has fundamentally changed who can build software and how fast it gets built. The $4.7 billion market, 92% developer adoption, and 2,400% search growth reflect a genuine paradigm shift.
