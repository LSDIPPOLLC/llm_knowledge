# Summary: Vibe Coding in 2026

## Core Definition
Vibe coding = describing what you want in plain language and letting AI generate code. Coined by Andrej Karpathy (Feb 2, 2025).

## Key Distinction
Simon Willison: "If an LLM wrote the code for you, and you reviewed, tested, and understood it — that's software development, not vibe coding."

## Key Numbers
- 92% of US developers use AI tools daily
- 41% of global code is AI-generated
- 60% of all new software will be AI-generated by end of 2026
- 63% of vibe coding users are non-developers
- ~24.7% of AI code has security flaws
- 1.7x more "major issues" vs human code

## Evolution: Vibe Coding → Agentic Engineering
Karpathy (2026): vibe coding is "passé". Agentic engineering = orchestrating teams of specialized agents with quality gates, not blindly accepting AI output.

## Best Practices
1. Always manually review auth/payments/infrastructure code
2. Write tests first, let AI implement to pass them
3. Use structured prompts with specific requirements
4. Commit frequently (AI can make sweeping changes)
5. Monitor token quota

## Tools
- **Agentic IDEs:** Google Antigravity (multi-agent parallel), Cursor (VS Code-based), Copilot (deep GitHub integration)
- **Terminal Agents:** Claude Code (high context), Codex CLI (Rust-based), Gemini CLI (free tier)

## Security Risks
- Hardcoded secrets in AI-generated code
- SQL injection / input validation failures
- Deprecated libraries reintroduced
- Hallucinated dependencies (typosquatting)
- Buffer overflows in C/C++

## Key Insight
Vibe coding = prototyping superpower. Agentic engineering = production workflow. Use appropriately.
